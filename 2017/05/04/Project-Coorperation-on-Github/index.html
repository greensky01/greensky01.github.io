Project Coorperation on Github

发表于 2017-05-04   |   分类于 学习   |  

团队成员角色：

project leader (PL)
project participants (PP)

平台搭建与流程

PL创建main repo（称为A），添加指定的PP作为collaborator，从而PL和被指定的PP都有了对A的“写”权限。

A的master分支即为project的稳定可交付版本（major release，1.0，2.0…），A的develop分支即为project的新功能发布分支（minor release，0.1 0.2 …），develop分支遇到整数版本号时，合并到master中。A的master和develop分支为保留分支，任何成员不能通过直接添加修改的方式改变分支内容，只能通过pull request进行修改。

PL在A中工作，创建属于自己的工作分支，最好使用前缀，以避免与可能的项目保留分支重名。

PP fork A，clone至本地进行工作，分支命名没有特殊要求，建议google “git 分支管理”。

团队中的任何成员，通过pull request向A的develop分支添加代码，或修复bug。每一个pull request，PL指定reviewer进行code review，当所有reviewer同意merge时，才能进行merge，否则提交pull request的成员，需要按照reviewer的意见，进行修改。针对一个pull request的讨论可以在github的web端完成。
